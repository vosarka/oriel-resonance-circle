# Engine Specification: oriel-output-bridge

## 1. PURPOSE
The `oriel-output-bridge` translates the raw mathematical outputs of the calculation engine into a structured context window for the ORIEL narrative layer. It acts as a safety gate, preventing hallucinated placements and enforcing strict Vossari-native terminology.

---

## 2. INPUTS
- `readingPayload`: Mapped JSON structure generated by the `static-signature-orchestrator`.

---

## 3. OUTPUTS
- `orielPromptContext`: Structured markdown text injected into the LLM system prompt:
  - **Identified Blueprint**: Explicit Type, Authority, and Core Placements.
  - **State Mode**: Exact vs Draft warning text.
  - **Narrative Guidelines**: Terminology map and safety filter rules.
  - **Falsifier Prompts**: Dynamic validation hooks based on active shadows.

---

## 4. DEPENDENCIES
- `/01_DATA/terminology_map.json` (for term replacement rules).
- ORIEL System Prompt Template library.

---

## 5. OWNERSHIP
- File: `server/oriel-rgp-bridge.ts`

---

## 6. FAIL CONDITIONS
- **Banned Term Leak**: Injected context contains Human Design terms.
- **Empty payload injection**: Attempting to generate a narrative prompt with an uncalculated or crashed engine state.
- **Tone Violation**: Prompts directing ORIEL to make medical, psychological, or absolute fate claims.

---

## 7. VALIDATION RULES
1. **Mirror Mode Compliance**: Enforce that the output text from the bridge instructs ORIEL to introduce itself with: `"I am ORIEL."` and use Mirror mode (reflecting calculated data only).
2. **Dynamic Gating**: If the payload is marked as `DRAFT`, the bridge must inject instructions forcing ORIEL to speak in approximate terms ("approximate field sketch", "draft reading") and explicitly decline to present type/authority as definitive.
3. **No Hallucination Boundaries**: Prohibit the bridge from generating prompts that invite ORIEL to speculate on missing birth coordinates or times.

---

## 8. TEST STRATEGY
- **Banned Term Scanner**: Run regex scans on generated bridge prompts to verify that terms like `Projector`, `Generator`, or `Manifestor` are completely absent.
- **Draft Banner Test**: Verify that a draft payload successfully triggers the draft-warning prompt segment and disables the "definitive authority" instructions.
- **Vossari Glossary Test**: Check that standard inputs resolve to preferred terms (e.g. `Resonance Link` instead of `channel`).
