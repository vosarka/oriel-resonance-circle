# Engine Specification: report-builder-engine

## 1. PURPOSE
The `report-builder-engine` compiles the calculated Static Signature payload and the generated ORIEL narrative into a 15-page premium symbolic diagnostic PDF document. It ensures visual consistency, auditability, and page-layout limits.

---

## 2. INPUTS
- `readingPayload`: Structured data containing type, authority, centers, active links, and activations.
- `orielNarrativeText`: String (narrative generated by ORIEL).
- `themeOptions`: Object defining the typography, palette, and scale parameters.

---

## 3. OUTPUTS
- `pdfStream`: Binary PDF stream (buffered document file).
- `generationLogs`: Object containing compilation metadata (generation time, page count, validation hash).

---

## 4. DEPENDENCIES
- PDF generation libraries (PDFKit, Puppeteer, or HTML-to-PDF wrapper).
- Chart and diagram canvas renderers (D3/Canvas node modules).

---

## 5. OWNERSHIP
- File: `/10_REPORT_BUILDER/` (conceptual package / module).

---

## 6. FAIL CONDITIONS
- **Page Count Mismatch**: Document compiled is less than 15 pages or exceeds 15 pages.
- **Render Overlap**: Text overflows page boundaries.
- **Missing Audit Seal**: Cover or page 2 fails to render the calculation integrity stamp.

---

## 7. VALIDATION RULES
1. **Strict Page Map**: Enforce exact page-by-page rendering order (as defined in `PDF_SCHEMA.md`).
2. **Branding Constraints**: Ensure all text elements use Vossari-native terms (run terminology validation pass before rendering).
3. **Data Provenance**: Page 2 (Calculation Seal) must display coordinates, UTC timestamp, and design Sun longitude.

---

## 8. TEST STRATEGY
- **Page Count Assert Test**: Generate report and verify page count = exactly 15 pages.
- **Visual Grid Audit**: Run headless checks or image comparisons to verify that text does not overlap with diagrams.
- **Checksum Integration**: Assert that the checksum/audit hash on Page 15 matches the hash computed from inputs.
